# GLinet BE3600 一键 iStoreOS 风格化技术原理说明

本项目的目标是：在 **不刷机、不改底层分区、不更换固件** 的前提下，让 GL-iNet BE3600 在视觉和功能上尽可能接近 iStoreOS，同时保留官方固件的稳定性与可回退能力。

---

## 1. 基础原理：GL.iNet 固件本质是 OpenWrt

GL-iNet 官方固件基于 OpenWrt 构建，具备以下特性：

- 使用 `opkg` 作为包管理器
- 支持自定义软件源（feeds）
- 支持 LuCI Web 界面与主题
- 支持通过软件包扩展功能（如 iStore、AdGuardHome 等）

本项目正是利用这一点，通过“软件源 + 软件包 + 配置”的方式实现 iStoreOS 风格化，而不是刷入完整 iStoreOS 固件。

---

## 2. 核心思路：软改而非刷机

本项目的脚本主要做三件事：

1. **添加 iStoreOS 相关软件源**
2. **安装 iStoreOS 风格的 UI、主题、商店与插件**
3. **调整 GL 官方界面入口与默认主题，使整体体验接近 iStoreOS**

整个过程：

- 不写入固件镜像
- 不修改 U-Boot
- 不改动分区表
- 不替换内核

因此属于“软改 UI + 功能增强”，而非“刷机”。

---

## 3. 软件源与软件包

脚本会添加 iStore 相关的软件源，例如：

- iStore / LinkEase 仓库
- 其他第三方 LuCI 插件源

然后通过 `opkg` 安装：

- `luci-app-istore` / `luci-app-store`（iStore 商店）
- `luci-theme-argon`（iStoreOS 常用主题）
- 其他 UI 辅助插件与功能组件

这使得：

- 原本 GL 官方固件的 LuCI 能够加载 iStoreOS 风格的界面与功能
- 用户可以通过 iStore 安装更多插件（如 AdGuardHome、NAS 相关组件等）

---

## 4. UI 与主题层面的改造

脚本会：

- 安装 Argon 主题（紫色风格）
- 将 Argon 设置为默认主题
- 调整 LuCI 菜单结构，使其更接近 iStoreOS 的布局
- 隐藏或弱化 GL 官方首页中的某些按钮（例如“格式化”按钮）

这些改动让用户在 Web 界面上看到的是：

- iStoreOS 风格的主题
- iStore 商店入口
- 更接近 iStoreOS 的菜单结构

从而在视觉上“像 iStoreOS”。

---

## 5. 功能增强：风扇控制、AdGuardHome、UI 辅助插件等

脚本还会：

- 安装并配置风扇温控脚本，支持自定义启动温度
- 提供 AdGuardHome 的启用/关闭入口
- 安装个性化 UI 辅助插件（例如 VMatrices 提供的增强组件）

这些功能在 iStoreOS 中也较为常见，因此进一步增强了“iStoreOS 化”的体验。

---

## 6. 安全性与可回退性

由于本项目不刷机、不改分区、不改内核，因此具备以下优点：

- **可通过卸载软件包、恢复配置来回退**
- **可通过恢复出厂设置回到 GL 官方默认状态**
- **不会因为刷错固件导致设备变砖**

同时，本仓库通过 GitHub Actions 自动从上游同步脚本：

- 上游可访问且文件非空 → 才会同步
- 上游失效 / 空文件 / 404 → 跳过同步，不覆盖本地脚本
- 内容无变化 → 不提交
- 内容有变化 → 自动更新

这样可以保证：

- 本地脚本始终尽量跟随上游更新
- 即使上游失效，本仓库仍保留最后一个可用版本

---

## 7. 与“真正刷入 iStoreOS 固件”的区别

| 对比项           | 本项目（软改）                         | 刷入 iStoreOS 固件                     |
|------------------|----------------------------------------|----------------------------------------|
| 是否刷机         | 否                                     | 是                                     |
| 是否改分区       | 否                                     | 可能会                                 |
| 是否改内核       | 否                                     | 是                                     |
| 可回退性         | 高（恢复出厂或卸载软件包即可）         | 需重新刷回官方固件                     |
| 风险             | 低                                     | 中～高（视操作熟练度与固件兼容性而定） |
| 体验接近 iStoreOS | 高（UI + 商店 + 插件）                | 完整 iStoreOS 体验                     |

本项目的定位是：

> 在尽量不增加风险的前提下，让 BE3600 尽可能接近 iStoreOS 的使用体验。
